Clase 4: Clases y Objetos en Java

¡Hola, Developers! Bienvenidos al corazón de la Programación Orientada a Objetos (POO). En esta clase aprenderemos cómo estructurar el código mediante planos conceptuales llamados Clases, cómo darles vida creando Objetos, y cómo gobernar su ciclo de vida y estado a través de Constructores, Getters & Setters, y el método representativo toString().

El Plano y la Construcción: Clases y Objetos

Para programar bajo el paradigma de POO, primero debemos comprender cómo modelar el mundo real en código:

La Clase (El Plano 📐)

Una clase es una **plantilla o molde** que define las características (atributos) y los comportamientos (métodos) que tendrán las entidades creadas a partir de ella. No consume memoria para almacenar datos, solo describe cómo serán. 

// Definición básica de la clase Coche
public class Coche {
    String marca;
    String modelo;
    int anio;
}

El Objeto (La Instancia 🚗)

Es el ejemplar concreto creado a partir de la clase. Ocupa un lugar físico en la memoria del computador y posee un **estado propio** (sus variables tienen valores específicos). 

// Creación (Instanciación) de un objeto
Coche miAuto = new Coche();
miAuto.marca = "Toyota";
miAuto.modelo = "Yaris";
miAuto.anio = 2022;
💡 Analogía del Mundo Real: Imagina el plano de un arquitecto para construir una casa (Clase). El plano solo dice dónde irá la cocina o cuántos pisos tendrá. La casa construida con ladrillo y cemento, pintada de azul en una calle real, es el Objeto. Puedes usar el mismo plano (Clase) para construir infinitas casas individuales (Objetos).

Inicialización de Estados: El Constructor

Un Constructor es un método especial que se invoca automáticamente con la palabra reservada new para inicializar el estado del objeto en el momento exacto de su nacimiento.

🛠️ Reglas del Constructor: 1. Debe tener **exactamente el mismo nombre** que la clase. 2. **No define ningún tipo de retorno**, ni siquiera void.

Usamos parámetros para obligar a que el objeto sea creado con sus datos iniciales. La palabra clave this hace referencia a los atributos de la instancia actual para diferenciarlos de las variables locales o parámetros: 

public class Coche {
    private String marca;
    private String modelo;

    // Constructor con parámetros
    public Coche(String marca, String modelo) {
        this.marca = marca;   // this.marca apunta al atributo de la clase
        this.modelo = modelo; // modelo apunta al parámetro recibido
    }
}

Si no escribes ningún constructor en tu clase, el compilador de Java inserta automáticamente un constructor vacío por defecto sin parámetros. Sin embargo, si creas al menos un constructor personalizado, el compilador **no** insertará el constructor por defecto, por lo que si necesitas uno vacío deberás declararlo explícitamente.

Encapsulamiento: Getters y Setters

El principio de **encapsulamiento** indica que los datos de un objeto deben protegerse del acceso y modificación directa por parte de agentes externos. Por ello, declaramos los atributos como private y proveemos métodos públicos:

🔑 Getter

Método de acceso público que retorna el valor de un atributo privado.

public String getMarca() {
    return this.marca;
}

⚙️ Setter

Método modificador público que permite cambiar el valor de un atributo privado bajo validación de reglas de negocio.

public void setAnio(int anio) {
    if (anio > 1886) { // Año de invención del primer auto
        this.anio = anio;
    } else {
        System.out.println("Año inválido");
    }
}
🛡️ ¿Por qué es importante? Si dejamos los atributos como públicos (public int anio;), cualquier parte de la aplicación podría asignar un año absurdo como miCoche.anio = -500;. Al forzar el uso del Setter, podemos validar y denegar valores inconsistentes.

Representación en Texto: El Método toString()

Por defecto, si intentas imprimir un objeto directamente (System.out.println(miCoche);), Java imprimirá su firma de referencia de memoria (por ejemplo, Coche@7a81197d). Esto no es amigable para los desarrolladores.

Sobrescribir toString()

Para resolver esto, sobrescribimos (@Override) el método heredado de la clase base Object, retornando una cadena estructurada que muestre los valores reales del objeto: 

@Override
public String toString() {
    return "Coche{" +
            "marca='" + marca + '\'' +
            ", modelo='" + modelo + '\'' +
            ", anio=" + anio +
            '}';
}

🛠️ Simulador: Instanciación e Inicialización

Modifica los campos del formulario para observar cómo cambia el código Java de instanciación y la representación en texto del objeto:

🔧 Invocación del Constructor en Main

Coche miCoche = new Coche("Tesla", "Model 3", 2024, 38990.00);

🖥️ Salida de miCoche.toString()

Coche{marca='Tesla', modelo='Model 3', anio=2024, precio=38990.00}
