Clase 2: Programación Orientada a Objetos.

La Programación Orientada a Objetos es un paradigma que estructura el código imitando elementos del mundo real. A través de sus cuatro pilares esenciales, aprenderás a construir software escalable, modular y reutilizable.

Clases y Objetos: El Molde y la Instancia

En el corazón de la POO se encuentran las entidades que representan conceptos del mundo real mediante software.

📋 La Clase

Es una plantilla o molde abstracto que define las propiedades y comportamientos comunes que tendrán los objetos creados a partir de ella. No consume memoria para datos reales.

📦 El Objeto

Es una instancia real y concreta de una clase. Posee un estado propio (sus variables tienen valores específicos) y puede ejecutar las acciones definidas en su molde.

⚙️ Atributos y Métodos

Los atributos son las características o datos del objeto (variables). Los métodos representan las acciones o funciones que el objeto sabe realizar.

Ejemplo: Creación de una clase e instanciación

public class Pokemon {
    // Atributos (Estado)
    String nombre;
    int nivel;

    // Constructor (Inicializador)
    public Pokemon(String nombre, int nivel) {
        this.nombre = nombre;
        this.nivel = nivel;
    }

    // Método (Comportamiento)
    public void atacar() {
        System.out.println(nombre + " lanza un ataque!");
    }
}

// Uso en el método main:
Pokemon miPokemon = new Pokemon("Pikachu", 15);
miPokemon.atacar(); // Imprime: Pikachu lanza un ataque!
¿Qué es el operador 'new' y el constructor?
  • new: Es la palabra clave que reserva espacio en la memoria Heap para almacenar el nuevo objeto.
  • Constructor: Un método especial que se llama automáticamente al usar new. Tiene el mismo nombre que la clase y sirve para inicializar los atributos.
💡 Buena práctica: En Java, las clases siempre siguen la convención UpperCamelCase (ej. GestorUsuarios), mientras que los objetos y métodos usan lowerCamelCase (ej. miCuenta).

Abstracción y Encapsulamiento

Estos dos pilares se enfocan en la protección de los datos y en simplificar la interacción con objetos complejos.

🎨 Abstracción

Consiste en aislar y extraer únicamente las características esenciales de un objeto para el dominio del problema, ignorando los detalles internos innecesarios.

🛡️ Encapsulamiento

Significa reunir los datos y los métodos que los manipulan en una "cápsula", restringiendo el acceso directo desde el exterior para evitar corromper el estado del objeto.

🔑 Modificadores de Acceso

Palabras clave que regulan la visibilidad: private (solo la propia clase), public (cualquiera) y protected (clases hijas).

Implementación con Getters y Setters

public class CuentaBancaria {
    private double saldo; // Nadie puede modificarlo directamente

    // Getter: Acceso seguro al valor
    public double getSaldo() {
        return this.saldo;
    }

    // Setter: Modificación controlada con validaciones
    public void depositar(double monto) {
        if (monto > 0) {
            this.saldo += monto;
        }
    }
}
⚠️ Riesgo de diseño: Exponer todos tus atributos con setters automáticos rompe el encapsulamiento. Modifica el estado del objeto únicamente a través de métodos de negocio con lógica válida.

Herencia: Reutilización Eficiente

Mecanismo que permite a una clase nueva adquirir las propiedades y métodos de una clase ya existente, promoviendo la reutilización del código.

🔝 Súperclase (Padre)

Clase genérica que contiene los atributos y comportamientos comunes. Ejemplo: Vehiculo.

🔜 Subclase (Hijo)

Clase especializada que hereda del padre utilizando la palabra clave extends. Puede añadir sus propios atributos o métodos.

🧬 Palabra clave 'super'

Se utiliza dentro de la subclase para invocar explícitamente al constructor o a los métodos de la súperclase padre.

Código de Herencia en Java

class Vehiculo {
    String marca;
    public void arrancar() { System.out.println("En marcha..."); }
}

// Coche hereda de Vehiculo
class Coche extends Vehiculo {
    int puertas;
    public void abrirMaletero() { System.out.println("Maletero abierto."); }
}

// Uso:
Coche miCoche = new Coche();
miCoche.marca = "Toyota"; // Atributo heredado
miCoche.arrancar();      // Método heredado

Polimorfismo: Muchas Formas

El polimorfismo es la capacidad de tratar dinámicamente a objetos de distintas subclases como si fueran de la súperclase, respondiendo de maneras distintas según su naturaleza real.

🎭 Sobreescritura (@Override)

Permite a una subclase modificar la implementación de un método heredado del padre para adaptarlo a su propia necesidad.

🔄 Enlace Dinámico

La JVM determina en tiempo de ejecución a qué método llamar basándose en el tipo real del objeto instanciado, no en el tipo de variable de referencia.

Demostración práctica de Polimorfismo

abstract class Animal {
    public abstract void emitirSonido();
}

class Perro extends Animal {
    @Override
    public void emitirSonido() { System.out.println("¡Guau!"); }
}

class Gato extends Animal {
    @Override
    public void emitirSonido() { System.out.println("¡Miau!"); }
}

// Aplicación de Polimorfismo
Animal miAnimal = new Perro();
miAnimal.emitirSonido(); // Imprime "¡Guau!"

miAnimal = new Gato();
miAnimal.emitirSonido(); // Imprime "¡Miau!"
