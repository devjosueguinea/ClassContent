Entender estas herramientas es crucial porque, antes de conectarnos a bases de datos reales con Spring Boot, simularemos bases de datos en memoria para probar nuestros Controllers y Services.

Colecciones Clave-Valor: Mapas en Java

La interfaz Map modela una estructura donde cada elemento se compone de una **Clave única** asociada a un **Valor**. No puede haber claves duplicadas.

⚡ HashMap

Almacena los elementos en una tabla hash. **No garantiza ningún orden** en particular para sus claves. Ofrece el mejor rendimiento para operaciones básicas de inserción, búsqueda y eliminación (tiempo constante O(1)). 

Map<Long, String> usuarios = new HashMap<>();
usuarios.put(101L, "Carlos");
usuarios.put(102L, "Ana");
usuarios.put(101L, "Juan"); // Reemplaza a "Carlos"

🌳 TreeMap

Implementa la interfaz NavigableMap utilizando un árbol Red-Black. **Ordena automáticamente las claves** según su orden natural (alfabético o numérico) o mediante un comparador personalizado. Tiene un rendimiento O(log n). 

Map<String, Double> precios = new TreeMap<>();
precios.put("Manzana", 1.5);
precios.put("Banana", 0.8);
precios.put("Cereza", 3.0);
// Se almacena ordenado: Banana, Cereza, Manzana

Colecciones de Elementos Únicos: Conjuntos (Set)

Un Set es una colección que **no permite elementos duplicados**. Es ideal cuando necesitas validar unicidad (como un registro de correos electrónicos o de IDs).

🚫 HashSet

Basado en un HashMap interno. No permite duplicados y **no garantiza orden** en absoluto. Es extremadamente eficiente para comprobar si un elemento ya existe en el conjunto. 

Set<String> correos = new HashSet<>();
correos.add("juan@mail.com");
correos.add("maria@mail.com");
correos.add("juan@mail.com"); // Duplicado, es ignorado

🧩 Regla de Unicidad

Para que un Set detecte duplicados correctamente al usar objetos personalizados (como una clase Estudiante), es obligatorio implementar correctamente los métodos equals() y hashCode() en dicha clase.

Repaso de Estructuras Secuenciales: List

A diferencia de los Conjuntos, las **Listas** admiten elementos duplicados y conservan un **estricto orden de inserción**.

📦 ArrayList

Estructura contigua en memoria. Ideal si vas a **leer o buscar elementos por su índice** muy seguido. Penaliza las inserciones en las primeras posiciones debido al desplazamiento de memoria.

🔗 LinkedList

Estructura de nodos doblemente enlazados en memoria. Ideal para **estructuras que cambian constantemente de tamaño** en sus extremos o donde realizas muchas inserciones y eliminaciones al principio.

Bases de Datos en Memoria para APIs con Spring Boot

Al construir una API RESTful con Spring Boot, solemos tener un Controlador (Controller) y un Servicio (Service). Antes de conectar una base de datos real (como MySQL u Oracle), simulamos la persistencia en memoria usando un HashMap dentro de nuestro Servicio.

Ejemplo de Servicio Simulado en Spring Boot

@Service
public class EstudianteService {
    // Simulamos la tabla de la base de datos en memoria
    private final Map<Long, Estudiante> baseDatos = new HashMap<>();
    private long idAutoincremental = 1;

    // CREATE / Guardar
    public Estudiante guardar(Estudiante estudiante) {
        if (estudiante.getId() == null) {
            estudiante.setId(idAutoincremental++);
        }
        baseDatos.put(estudiante.getId(), estudiante);
        return estudiante;
    }

    // READ / Obtener todos
    public List<Estudiante> obtenerTodos() {
        return new ArrayList<>(baseDatos.values()); // Convertimos los valores del mapa a una lista
    }

    // READ / Obtener por ID
    public Estudiante obtenerPorId(Long id) {
        return baseDatos.get(id);
    }

    // DELETE / Eliminar
    public void eliminar(Long id) {
        baseDatos.remove(id);
    }
}
