¿Qué es una API RESTful?

Una API (Application Programming Interface) es un conjunto de reglas, protocolos y herramientas que permite que diferentes sistemas o componentes de software se comuniquen entre sí, actuando como un intermediario que facilita el intercambio seguro de datos.

Componentes Clave de una Petición REST

📍 Endpoint

URL única encargada de identificar un recurso específico en el servidor.
Ejemplo: /api/v1/usuarios

🔀 Métodos HTTP

Verbos que indican la acción a realizar:

  • GET: Obtener datos.
  • POST: Crear nuevos datos.
  • PUT / PATCH: Actualizar datos.
  • DELETE: Eliminar datos.

📇 Headers & Body

Headers: Metadatos de la petición (ej. Content-Type: application/json).

Body: Datos enviados o recibidos estructurados comúnmente en formato JSON.

Arquitectura del Software en Spring Boot

Para construir APIs mantenibles y escalables, estructuramos nuestro backend dividiendo el código en capas con responsabilidades únicas y bien delimitadas:

⚡ Controllers (EndPoints / Rutas HTTP)
⚙️ Services (Lógica de Negocio y Validaciones CRUD)
🗄️ Repositories (Acceso a Datos - Interfaces JPA)
📦 Entities (Representación fiel de las Tablas de la BD)
💡 Selecciona o pasa el cursor sobre las capas superiores para ver sus componentes auxiliares adicionales (Config, Exceptions, DTOs/Models, Utils).

Estructura Completa de Paquetes

Capa / Directorio Propósito Primario dentro del Proyecto
config Configuraciones globales de seguridad, CORS o Beans externos.
controllers Recibe las peticiones HTTP y despacha las respuestas estructuradas.
services Implementa las reglas del negocio, orquestación de operaciones y validación de datos.
repositories Interfaces de Spring Data (JPA, JDBC) para interactuar directamente con la BD.
entities Clases que mapean de uno a uno las tablas relacionales de la base de datos.
exceptions Interceptores globales encargados del manejo controlado de errores del sistema.
models Estructuras de transferencia de datos como DTOs y formateadores de ApiResponse.
utils Clases utilitarias con métodos estáticos de soporte común.
⚠️ Fichero application.properties: Ubicado en los recursos del proyecto, almacena de forma centralizada todas las variables críticas de conexión de la BD y puertos del servidor.

El Poder de las Anotaciones en Spring Boot

Las anotaciones actúan como metadatos decorativos que le indican al contenedor de Spring cómo instanciar, inyectar y gestionar el comportamiento de nuestras clases en tiempo de ejecución.

1. Definición de Beans y Componentes Estándar
  • @Component: Marca una clase genérica como componente administrado por el contenedor IoC.
  • @Service: Especialización de componente para alojar exclusivamente lógica empresarial.
  • @Repository: Indica acceso a persistencia de datos (gestiona excepciones SQL nativas).
  • @Controller / @RestController: Habilita la clase para interceptar peticiones web de tipo MVC o RESTful JSON.
  • @Configuration / @Bean: Define métodos de instanciación manual para librerías de terceros.
2. Inyección de Dependencias Automática
  • @Autowired: Enlaza e inyecta de forma automática una dependencia compatible por su tipo.
  • @Qualifier: Rompe la ambigüedad indicando el ID exacto del Bean que deseas inyectar si existen múltiples implementaciones.
  • @Value: Extrae e inyecta parámetros literales guardados en tu application.properties.
3. Enrutamiento Web Spring MVC / REST
  • @RequestMapping: Configura el prefijo de ruta URL base sobre toda la clase controladora.
  • @GetMapping / @PostMapping / @PutMapping / @DeleteMapping: Mapean verbos HTTP directos.
  • @PathVariable: Extrae variables incrustadas directamente en el segmento de la URL (ej: /usuarios/{id}).
  • @RequestParam: Lee query parameters tradicionales de filtrado (ej: ?page=1&limit=10).
  • @RequestBody: Deserializa el cuerpo JSON entrante convirtiéndolo instantáneamente en un Objeto Java (POJO).
4. Mapeo ORM con Spring Data JPA
  • @Entity: Configura la clase Java para ser tratada como un modelo relacional de JPA.
  • @Table: Asigna explícitamente el nombre físico de la tabla en el motor SQL de destino.
  • @Id: Delimita qué propiedad actuará como la llave primaria (Primary Key).
  • @GeneratedValue: Determina estrategias de autoincremento (Identity, Sequence, etc.).

Reglas Obligatorias del Proyecto

// 📜 Directrices estricta de desarrollo limpio:
1. Capa Service: Todo archivo de servicio requiere llevar imperativamente la anotación @Service.
2. Capa Repository: Las interfaces hijas de JpaRepository deben poseer la anotación @Repository.
3. Capa Controller: Deben configurarse obligatoriamente con las anotaciones @RestController y @RequestMapping.
4. Entidades JPA: Deben incluir sus decoradores bases: @Entity @Getter @Setter @Id @ToString @EqualsAndHashCode.
5. Modelos DTO: Están prohibidas las lógicas internas; únicamente deben contener campos y anotaciones de validación.

Gestión de Proyectos Mediante Maven

Una dependencia representa cualquier módulo o librería externa empaquetada que integramos al proyecto para evitar programar código recurrente desde cero.

💡 El archivo pom.xml (Project Object Model): Es el manifiesto principal escrito en formato XML que reside en la raíz del proyecto. Le indica a Maven la configuración del entorno, compilación y las coordenadas de dependencias externas a descargar automáticamente desde los repositorios centrales.

Coordenadas Universales GAV

Maven rastrea e identifica con precisión quirúrgica cualquier librería del mundo mediante tres identificadores jerárquicos:

  • Group ID (groupId): El namespace corporativo u organizacional (ej. org.springframework.boot).
  • Artifact ID (artifactId): El nombre específico del módulo o subproyecto compilado (ej. spring-boot-starter-web).
  • Version: La release exacta del artefacto utilizado (ej. 3.2.4).

Ejemplo Real de Bloque de Dependencia

<!-- Fichero pom.xml - Inclusión del ecosistema web de Spring -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
