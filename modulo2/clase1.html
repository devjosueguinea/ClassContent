Clase 1: Fundamentos de Programación en Java

En esta clase introductoria recorreremos los pilares esenciales de la programación antes de adentrarnos en el paradigma orientado a objetos. Dominar estos conceptos es indispensable para construir soluciones robustas y legibles.

Fundamentos de Programación

La programación es el arte de dar instrucciones precisas a una computadora para resolver problemas. Antes de escribir código, debemos comprender los conceptos base que sostienen cualquier lenguaje.

📋 Algoritmo

Secuencia finita, ordenada y no ambigua de pasos que transforman una entrada en una salida deseada. Es la solución abstracta al problema, independiente del lenguaje.

☕ Java y la JVM

Java compila el código fuente (.java) a bytecode (.class). La JVM (Java Virtual Machine) interpreta ese bytecode en cualquier sistema operativo — principio "Write Once, Run Anywhere".

📦 Variables y Constantes

Una variable es un espacio en memoria con nombre y tipo que puede cambiar. Una constante se declara con final y su valor no puede modificarse tras su inicialización.

Estructura mínima de un programa Java

// Clase principal — el nombre debe coincidir con el archivo
public class HolaMundo {
    // Punto de entrada obligatorio de todo programa
    public static void main(String[] args) {
        System.out.println("¡Hola, Mundo!");
    }
}
¿Por qué public static void main?
  • public: accesible desde cualquier clase, necesario para que la JVM lo invoque.
  • static: pertenece a la clase, no a una instancia; la JVM no necesita crear un objeto para llamarlo.
  • void: no devuelve ningún valor.
  • String[] args: arreglo de argumentos enviados desde la línea de comandos.
💡 Buena práctica: Usa nombres descriptivos en inglés o español consistentemente. El código se lee muchas más veces de las que se escribe.

Tipos de Datos en Java

Declaración e inicialización

// Declarar y asignar en la misma línea
int edad = 21;
double promedio = 8.75;
boolean aprobado = true;
char inicial = 'J';
String nombre = "Josué";

// Constante — no puede cambiar después de inicializar
final double PI = 3.14159;
⚠️ Conversión de tipos (casting): al asignar un tipo mayor a uno menor debes indicarlo explícitamente: int n = (int) 3.99; → n vale 3.
Autoboxing y Wrapper Classes
Java dispone de clases envolventes (wrappers) para cada tipo primitivo: Integer, Double, Boolean, etc. El autoboxing convierte automáticamente entre primitivo y wrapper cuando es necesario, por ejemplo al guardar un int en un ArrayList<Integer>.

Operadores: Aritméticos, Relacionales y Lógicos

Los operadores son los símbolos que le indican al compilador qué operaciones realizar sobre los operandos. Java los clasifica en tres grandes grupos que es indispensable dominar.

🔢 Aritméticos

Realizan operaciones matemáticas entre valores numéricos.

+ Suma   - Resta

* Multiplicación   / División

% Módulo (residuo)

++ Incremento   -- Decremento

⚖️ Relacionales

Comparan dos valores y devuelven un boolean.

== Igual a

!= Diferente de

> Mayor que   < Menor que

>= Mayor o igual   <= Menor o igual

🧠 Lógicos

Combinan expresiones booleanas para formar condiciones compuestas.

&& AND — ambas verdaderas

|| OR — al menos una verdadera

! NOT — niega el valor

^ XOR — exactamente una verdadera

Ejemplos en Java

int a = 10, b = 3;
System.out.println(a + b); // 13 — Aritmético
System.out.println(a % b); // 1 — Módulo
System.out.println(a > b); // true — Relacional
System.out.println(a == b); // false

boolean esMayor = (a > 5) && (b < 10);
System.out.println(esMayor); // true — Lógico AND
Precedencia de operadores
Java evalúa los operadores en este orden (de mayor a menor prioridad):

  1. ++ -- (postfijo y prefijo)
  2. * / %
  3. + -
  4. < > <= >=
  5. == !=
  6. &&
  7. ||
  8. = (asignación)
Usa paréntesis para forzar el orden que necesites.

Estructuras Selectivas y Repetitivas

Las estructuras de control dirigen el flujo de ejecución del programa, permitiendo tomar decisiones y repetir bloques de código de forma controlada.

Estructuras Selectivas (Condicionales)

🔀 if / else if / else

Evalúa una condición booleana y ejecuta uno de varios bloques según el resultado.

🔀 switch-case

Compara el valor de una variable contra múltiples casos constantes. Más limpio que múltiples if cuando hay muchas opciones fijas.

🔀 Operador ternario

Expresión compacta: condición ? valorSi : valorNo. Ideal para asignaciones simples en una sola línea.

// if / else if / else
int nota = 75;
if (nota >= 90) {
    System.out.println("Excelente");
} else if (nota >= 70) {
    System.out.println("Aprobado");
} else {
    System.out.println("Reprobado");
}

// switch-case
int dia = 2;
switch (dia) {
    case 1: System.out.println("Lunes"); break;
    case 2: System.out.println("Martes"); break;
    default: System.out.println("Otro día");
}

// Ternario
String estado = (nota >= 70) ? "Aprobado" : "Reprobado";

Estructuras Repetitivas (Bucles)

🔄 for

Ideal cuando el número de iteraciones es conocido. Inicialización, condición e incremento en una misma línea.

🔄 while

Se ejecuta mientras la condición sea verdadera. Puede no ejecutarse si la condición es falsa desde el inicio.

🔄 do-while

Ejecuta el cuerpo al menos una vez y luego verifica la condición. Útil para menús y validaciones de entrada.

// for — imprime del 1 al 5
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}

// while
int n = 10;
while (n > 0) {
    System.out.println(n);
    n--;
}

// do-while — útil para menús
int opcion;
do {
    // mostrar menú y capturar opcion...
    opcion = 0; // simulación
} while (opcion != 0);
break y continue — control fino del bucle
  • break: Sale inmediatamente del bucle o bloque switch actual.
  • continue: Salta el resto del cuerpo de la iteración actual y avanza a la siguiente.
for (int i = 1; i <= 10; i++) {
    if (i == 5) continue; // salta el 5
    if (i == 8) break; // detiene en 8
    System.out.println(i);
}
