Clase 3: Vectores y Listas en Java

¡Hola, Developers! En esta clase exploraremos cómo Java maneja las colecciones de datos lineales de forma estática y dinámica. Comprenderemos los clásicos Arrays (vectores), analizaremos el histórico Vector, y dominaremos las implementaciones modernas de la interfaz List: ArrayList y LinkedList.

Estructuras Contiguas: Arrays Estáticos y la clase Vector

En Java, el Array (o Arreglo/Vector) es la estructura de datos más básica y nativa para almacenar secuencias de elementos. Sin embargo, tiene una restricción crítica: su tamaño es fijo una vez creado.

📏 Tamaño inmutable: Cuando declaras int[] numeros = new int[5];, el sistema reserva exactamente espacio en memoria para 5 enteros. No puedes expandirlo dinámicamente a 6 sin crear un nuevo array y copiar los datos manualmente.

Ejemplo básico de inicialización y asignación de valores en un array:

// Declaración e inicialización con tamaño definido
int[] edades = new int[3];
edades[0] = 18;
edades[1] = 25;
edades[2] = 30;

// Inicialización abreviada con valores implícitos
String[] nombres = {"Ana", "Juan", "Pedro"};

Introducido en Java 1.0, Vector es una clase que representa un array dinámico. A diferencia de las colecciones modernas, es sincronizado (thread-safe). Esto significa que todos sus métodos de acceso están protegidos internamente para entornos multi-hilo.

⚠️ ¿Por qué ya no se recomienda?

  • La sincronización automática añade una sobrecarga de rendimiento muy alta para aplicaciones single-thread.
  • En desarrollo moderno, preferimos ArrayList y si necesitamos control de concurrencia, usamos colecciones especializadas como CopyOnWriteArrayList o bloques de sincronización explícitos.
// Uso clásico de Vector (Legacy)
Vector<String> vectorLegacy = new Vector<>();
vectorLegacy.add("Java 1.0");
vectorLegacy.add("Legacy Code");

El Array Dinámico por Excelencia: ArrayList

ArrayList es la clase más utilizada del Java Collections Framework. Implementa la interfaz List y resuelve la limitación del tamaño de los arrays estándar mediante un mecanismo de **redimensionamiento dinámico**.

¿Cómo funciona internamente?

Usa un array estático común por debajo. Cuando este array se llena (alcanza el 100% de su capacidad), ArrayList crea automáticamente un nuevo array con aproximadamente **1.5 veces el tamaño del original** y transfiere todos los elementos.

Código de Ejemplo Común

import java.util.ArrayList;
import java.util.List;

public class EjemploListas {
    public static void main(String[] args) {
        // Declaración recomendada usando la interfaz List
        List<String> tareas = new ArrayList<>();
        
        // Agregar elementos
        tareas.add("Aprender POO");
        tareas.add("Estudiar Spring Boot");
        
        // Obtener por índice
        String primera = tareas.get(0);
        
        // Remover elementos
        tareas.remove(1);
        
        // Tamaño actual
        int total = tareas.size();
    }
}
💡 Buena práctica: Declara la variable con la interfaz List en lugar del tipo de implementación concreta (ej. List<T> lista = new ArrayList<>()). Esto facilita cambiar de implementación más tarde sin romper el contrato del código.

Listas Doblemente Enlazadas: LinkedList

A diferencia de ArrayList, LinkedList no almacena sus datos en posiciones contiguas de memoria. Utiliza una estructura de **nodos**, donde cada elemento contiene el valor y dos punteros: uno al nodo anterior y otro al siguiente.

Ventajas de LinkedList

  • Inserciones y eliminaciones rápidas: Solo requiere reasignar los punteros de los nodos adyacentes. No necesita desplazar ningún elemento en memoria.
  • Funciona de forma nativa como estructura de **Cola (Queue)** o **Pila (Stack)** implementando interfaces como Deque.

Código de Ejemplo

import java.util.LinkedList;
import java.util.List;

public class EjemploLinkedList {
    public static void main(String[] args) {
        List<Integer> historial = new LinkedList<>();
        historial.add(200);
        historial.add(404);
        historial.add(500);
        
        // También puedes usar métodos de Queue si declaras como tal:
        LinkedList<String> cola = new LinkedList<>();
        cola.addFirst("Cliente 1");
        cola.addLast("Cliente 2");
    }
}

Manipulación y Recorrido de Colecciones

Java ofrece múltiples maneras de recorrer (iterar) arrays y listas dinámicas según la versión de Java o la semántica del código.

Diferentes Formas de Iterar una Lista

List<String> lenguajes = List.of("Java", "Kotlin", "TypeScript");

// 1. Bucle For Clásico (Basado en Índice)
for (int i = 0; i < lenguajes.size(); i++) {
    System.out.println(lenguajes.get(i));
}

// 2. Bucle For-Each (Recomendado y legible)
for (String lenguaje : lenguajes) {
    System.out.println(lenguaje);
}

// 3. Expresiones Lambda / Método de Referencia (Java 8+)
lenguajes.forEach(System.out.println);

// 4. API de Streams (Para procesamiento funcional)
lenguajes.stream()
    .filter(l -> l.startsWith("J"))
    .forEach(System.out::println);

ArrayList vs LinkedList: Auditoría de Rendimiento

Selecciona una operación común para evaluar cuál estructura de datos ofrece un mejor desempeño computacional:

🔍 Búsqueda de Elemento por Índice (ej. lista.get(450)):

ArrayList (Ganador ⚡ - O(1)): Al ser un array contiguo en memoria, calcula la dirección exacta instantáneamente.
LinkedList (Lento - O(n)): Debe recorrer los nodos uno a uno desde el inicio (o el final) hasta llegar al índice 450.
