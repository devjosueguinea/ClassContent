🗄️ Persistencia de Datos

Sin datos, no hay negocio. Dominar Oracle Database implica entender transacciones ACID y optimización de queries complejas para manejar millones de registros. 

SELECT e.nombre, d.departamento 
FROM empleados e 
JOIN departamentos d ON e.id = d.id
WHERE e.rol = 'ARCHITECT';
