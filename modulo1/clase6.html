Clase 6: Relación de Herencia (Generalización)

La herencia es uno de los pilares fundamentales de la Programación Orientada a Objetos (POO). En el modelado UML, este concepto se conoce bajo la terminología técnica de Generalización. Describe una relación jerárquica de tipo "es un" (is-a) entre una clase más general (superclase) y una clase más específica (subclase).

Conceptos Fundamentales de la Herencia

La generalización permite estructurar las clases de manera que compartan atributos y comportamientos comunes, evitando la redundancia en el código y facilitando la mantenibilidad.

Superclase (Clase Padre)

Es la entidad que concentra los atributos y métodos comunes. Puede ser una clase concreta o una clase abstracta (que no se puede instanciar directamente y sirve únicamente como plantilla).

Subclase (Clase Hija)

Es la entidad especializada. Hereda automáticamente los miembros visibles de la superclase y puede agregar sus propios atributos y operaciones únicos, o sobrescribir los existentes.

Símbolo de Representación

Se representa mediante una línea sólida continua que finaliza en una punta con forma de triángulo vacío y cerrado, el cual apunta directamente hacia la superclase.

Detalles Técnicos Avanzados

Despliega los siguientes temas para profundizar en el comportamiento de la herencia a nivel de compilación y diseño:

1. Modificadores de Acceso y Visibilidad (#)
Para permitir que una subclase acceda a los atributos de la superclase sin exponerlos al exterior del objeto, se utiliza la visibilidad protegida (protected), representada con el símbolo de almohadilla (#). Si fueran privados (-), la subclase no podría acceder a ellos directamente.
2. Sobrescritura de Métodos (Overriding)
La sobrescritura ocurre cuando una subclase proporciona una implementación específica de un método que ya está definido en su superclase. En Java, esto se documenta y valida mediante la anotación @Override, garantizando el polimorfismo dinámico.
3. Herencia Múltiple y Simulación
En UML se permite que una clase herede de más de una superclase. Sin embargo, lenguajes como Java prohíben la herencia múltiple directa de clases para evitar la ambigüedad (el "problema del diamante"). Java solventa esto permitiendo la herencia simple de clases combinada con la implementación múltiple de Interfaces.

Conceptos Complementarios: Constructor y toString()

Para dominar la implementación práctica de la herencia en lenguajes como Java, es fundamental comprender el rol de los constructores y la representación textual de los objetos:

El Constructor

Es un método especial encargado de inicializar el estado de un objeto cuando se crea con la palabra clave new. En herencia, la subclase debe invocar al constructor de la superclase utilizando la instrucción super() en su primera línea, garantizando que los atributos heredados se inicialicen correctamente.

Método toString()

Heredado de la superclase raíz de Java (Object). Por defecto, imprimir un objeto muestra su tipo y hashcode (ej. Vehiculo@4f2a7a). Al sobrescribir este método, definimos una cadena de texto personalizada y legible con los atributos del objeto, lo cual es de vital importancia para la depuración y auditoría (debugging) de software.

Traducción de UML a Código (Java)

Así es como se traduce el diagrama gráfico anterior en código de programación estructurado en Java, incluyendo constructores y la sobrescritura de toString():

// Definition of Abstract Superclass
public abstract class Vehiculo {
    protected String marca;
    protected double precio;

    public Vehiculo(String marca, double precio) {
        this.marca = marca;
        this.precio = precio;
    }

    public void arrancar() {
        System.out.println("Vehículo arrancando...");
    }

    @Override
    public String toString() {
        return "Vehiculo[Marca: " + marca + ", Precio: " + precio + "]";
    }

    public abstract void mostrarInfo();
}

// Subclass extending Superclass
public class Automovil extends Vehiculo {
    private int cantidadPuertas;

    public Automovil(String marca, double precio, int cantidadPuertas) {
        super(marca, precio);
        this.cantidadPuertas = cantidadPuertas;
    }

    @Override
    public void mostrarInfo() {
        System.out.println("Auto Marca: " + marca + ", Puertas: " + cantidadPuertas);
    }

    @Override
    public String toString() {
        return "Automovil[" + super.toString() + ", Puertas: " + cantidadPuertas + "]";
    }
}
