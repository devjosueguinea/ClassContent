Clase 5: Diagramas de Clase

El diagrama de clases es el corazón del modelado en el Lenguaje de Modelado Unificado (UML). Es un diagrama de estructura estática que describe la estructura de un sistema mostrando sus clases, sus atributos, operaciones (o métodos) y las relaciones entre los objetos.

¿Qué es una Clase en UML?

En términos sencillos, una clase es una plantilla o plano para crear objetos. En términos más técnicos, define un conjunto de objetos que comparten las mismas especificaciones de características, restricciones y semántica. Visualmente se representa mediante un rectángulo dividido en tres secciones:

Nombre de la Clase

Se coloca en la sección superior. Debe ser un sustantivo en singular y estar centrado. Se escribe en formato PascalCase (ej. GestorPrestamo).

Atributos

Se colocan en la sección intermedia. Representan las propiedades o datos de la clase. Deben llevar tipo de dato y nivel de visibilidad.

Operaciones o Métodos

Se colocan en la sección inferior. Representan los comportamientos y funciones de la clase, incluyendo sus parámetros y valor de retorno.

Niveles de Visibilidad (Encapsulamiento)

Los modificadores de acceso o visibilidad permiten proteger el estado interno del objeto, garantizando el encapsulamiento. En UML se utilizan los siguientes símbolos:

+ Público (Public)
Cualquier clase externa puede acceder al elemento. Se utiliza principalmente para métodos que forman la interfaz o servicio del objeto.
+ calcularTotal(): double
- Privado (Private)
Solo la propia clase que lo define puede acceder al atributo o método. Es la opción recomendada por defecto para todos los atributos.
- saldoInterno: double
# Protegido (Protected)
Solo la propia clase y sus subclases (herencia) pueden acceder al miembro. Es vital para permitir la extensión mediante especialización.
# aplicarDescuentoEspecial()
~ Paquete (Package / Default)
El elemento es visible solo para clases que se encuentran dentro del mismo paquete o carpeta de compilación.
~ verificarPaquete()

Relaciones en los Diagramas de Clase

Las clases rara vez trabajan solas. Las relaciones describen la forma en que las clases interactúan y cooperan entre sí.

Asociación

Asociación Simple

Es una relación estructural que describe una conexión bidireccional o unidireccional entre objetos (ej. Un Cliente realiza un Pedido).

Representación: Una línea sólida continua.

Agregación

Agregación ("Tiene un")

Relación todo-parte donde el ciclo de vida del objeto secundario NO depende del principal (ej. Una Universidad tiene Profesores. Si la Universidad cierra, los profesores siguen existiendo).

Representación: Línea sólida con un rombo vacío en el extremo del "todo".

Composición

Composición ("Es parte de")

Relación todo-parte fuerte donde la parte NO puede existir sin el todo. Si se destruye el todo, se destruye la parte (ej. Una Casa y sus Habitaciones).

Representación: Línea sólida con un rombo lleno en el extremo del "todo".

Herencia

Generalización / Herencia

Indica que una subclase hereda los atributos y métodos de una superclase (ej. Un Perro hereda de Animal).

Representación: Línea sólida con un triángulo vacío apuntando a la superclase.

Realización

Realización / Implementación

Ocurre cuando una clase concreta implementa las operaciones definidas por una Interfaz.

Representación: Línea punteada con un triángulo vacío apuntando a la interfaz.

Dependencia

Dependencia ("Usa un")

Una clase utiliza temporalmente a otra como parámetro o variable local en un método (ej. Una clase Factura usa una Impresora para imprimirse).

Representación: Línea punteada con una punta de flecha abierta.

Ejemplo Práctico: De UML a Código de Programación

A continuación se presenta un caso práctico donde traducimos un fragmento de diagrama de clase en código real de Java:

// 1. Definición de la Superclase
public abstract class Persona {
    protected string id;
    protected string nombre;

    public Persona(string id, string nombre) {
        this.id = id;
        this.nombre = nombre;
    }

    public abstract void MostrarInformacion();
}

// 2. Clase Hija (Herencia / Generalización)
public class Cliente : Persona {
    private string correo;

    public Cliente(string id, string nombre, string correo) : base(id, nombre) {
        this.correo = correo;
    }

    public override void MostrarInformacion() {
        Console.WriteLine($"[Cliente] ID: {id}, Nombre: {nombre}, Correo: {correo}");
    }
}
